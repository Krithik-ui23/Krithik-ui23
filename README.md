### What it does

| Feature | Description |
|:---|:---|
| 🚨 **Emergency SOS** | One-click alert that emails all nearby matching donors instantly via Gmail SMTP |
| 📍 **GPS Hospital Sorting** | Haversine formula sorts 10 real hospitals by distance from user's live location |
| 🗺️ **Live Map** | Leaflet.js map with real-time hospital, SOS alert and donor markers |
| 📊 **Blood Inventory** | Chart.js dashboard showing Critical / Low / Adequate stock per hospital |
| 🩸 **Eligibility Tracker** | 56-day WHO cooldown timer enforced server-side with animated progress bar |
| 📅 **Appointment Booking** | Book donation slots at real hospitals with GPS-sorted selection modal |
| 🔐 **Admin Portal** | Token-protected CRUD access to all users, alerts, donations and appointments |
| 📧 **Email Notifications** | Beautiful HTML emails sent in background threads with zero API latency |

| Feature | Description |
|:---|:---|
| 📊 **Real-Time Market Engine** | Continuously fetches and updates top 50 crypto assets with minimal latency |
| 📈 **Advanced Charting** | Multi-range interactive price visualization for trend analysis |
| ⭐ **Realtime Watchlist System** | Cross-tab synchronized watchlist powered by Supabase Realtime |
| 💼 **Portfolio Analytics** | Tracks investments, returns and portfolio performance dynamically |
| 🔍 **Optimized Search** | Debounced, fast search with instant result rendering |
| 🌙 **Persistent Theming** | System-aware dark mode with user preference override |
| 🔐 **Secure Auth Layer** | Role-based protected routes using Supabase authentication |
| ⚡ **Event-Driven Updates** | WebSocket-based live data sync without polling |
| 📡 **Server-Side API Layer** | Next.js route handlers act as a secure proxy + caching layer |

## 🏗️ How I Build Things

```
   IDEA  ──▶  DESIGN  ──▶  BUILD  ──▶  TEST  ──▶  SHIP
     │            │            │           │          │
  Problem      Sketch       Backend     Debug      GitHub
  first        the UI       routes      every    + README
               and DB       + DB        edge
               schema       tables      case
```

- 🔴 I start from **real-world problems** — LifeStream was born because blood banks lack real-time donor-patient connection
- 🔴 I design the **database schema first** before writing a single route
- 🔴 I build the **backend API** and test every endpoint before touching the frontend
- 🔴 I connect frontend to backend with `fetch()` and handle every error state
- 🔴 I secure **credentials in `.env`** and never push secrets to GitHub
